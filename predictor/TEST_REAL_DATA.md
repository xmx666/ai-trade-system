# krnos预测服务真实数据测试指南

## 📋 测试流程

### 步骤1: 获取真实币安数据

首先运行Go程序从币安API获取真实数据：

```bash
cd predictor
go run test_krnos_real.go
```

这个程序会：
1. 从币安API获取BTCUSDT的真实K线数据（450条3分钟K线）
2. 提取价格和成交量序列
3. 验证数据完整性
4. 保存到 `test_data.json` 文件

### 步骤2: 运行Python预测测试

获取真实数据后，运行Python测试脚本：

```bash
python3 test_krnos.py
```

这个脚本会：
1. 加载真实数据文件
2. 初始化预测器
3. 运行预测（120步）
4. 验证预测结果
5. 测试缓存功能

## 🔍 测试内容

### Go程序测试 (`test_krnos_real.go`)

1. **从币安获取真实数据**
   - 使用 `market.NewHistoryClient()` 获取历史K线
   - 获取450条3分钟K线数据
   - 验证数据完整性

2. **数据提取和验证**
   - 提取价格序列（收盘价）
   - 提取成交量序列
   - 验证数据长度和范围

3. **保存数据**
   - 保存为JSON格式
   - 供Python脚本使用

### Python脚本测试 (`test_krnos.py`)

1. **加载真实数据**
   - 从 `test_data.json` 加载数据
   - 验证数据格式和完整性

2. **预测功能测试**
   - 使用真实数据运行预测
   - 验证预测结果格式
   - 检查预测曲线长度（120步）

3. **其他功能测试**
   - 预测缓存
   - 重新预测判断
   - 数据验证

## 📊 标准测试参数

- **币种**: BTCUSDT
- **K线间隔**: 3m（3分钟）
- **历史数据步数**: 450步（标准）
- **预测步数**: 120步（标准）
- **蒙特卡罗模拟**: 10次（限制）

## ⚠️ 注意事项

1. **数据获取**
   - 需要网络连接访问币安API
   - 如果API限制，可能需要等待
   - 确保币安API可访问

2. **数据文件**
   - `test_data.json` 会在运行Go程序时生成
   - 如果文件已存在，Python脚本会使用现有数据
   - 可以删除文件重新获取最新数据

3. **测试环境**
   - 确保Go环境正常
   - 确保Python环境正常
   - 确保numpy已安装

## 🔧 故障排除

### 问题1: Go程序无法获取数据

**可能原因**:
- 网络问题
- 币安API限制
- 币种不存在

**解决**:
- 检查网络连接
- 等待一段时间后重试
- 验证币种符号

### 问题2: Python脚本找不到数据文件

**解决**:
```bash
# 先运行Go程序获取数据
go run test_krnos_real.go

# 然后运行Python脚本
python3 test_krnos.py
```

### 问题3: 数据不足

**可能原因**:
- 币安API返回数据少于请求数量
- 币种交易不活跃

**解决**:
- 检查币种是否正常交易
- 尝试其他币种（修改Go程序中的Symbol常量）
