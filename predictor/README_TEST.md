# krnos预测服务测试说明

## 🎯 测试目标

使用**真实币安数据**测试krnos预测服务，确保与真实运行环境一致。

## 📋 测试流程

### 步骤1: 获取真实币安数据

```bash
cd predictor
go run test_krnos_real.go
```

**功能**:
- 从币安API获取BTCUSDT的真实K线数据
- 获取450条3分钟K线（标准历史数据步数）
- 提取价格和成交量序列
- 验证数据完整性
- 保存到 `test_data.json` 文件

**输出**:
- `test_data.json`: 包含真实价格和成交量数据的JSON文件

### 步骤2: 运行预测测试

```bash
python3 test_krnos.py
```

**功能**:
- 加载真实数据文件
- 初始化预测器
- 运行预测（120步，标准预测步数）
- 验证预测结果
- 测试缓存功能

## ✅ 验证要点

1. **数据真实性**: ✓ 从币安API获取，不是模拟数据
2. **数据完整性**: ✓ 确保有450条数据
3. **数据新鲜度**: ✓ 数据是最新的
4. **预测准确性**: 需要与实际市场对比验证

## 📊 标准参数

- **历史数据**: 450步（标准）
- **预测步数**: 120步（标准）
- **蒙特卡罗模拟**: 10次（限制）
- **测试币种**: BTCUSDT
- **K线间隔**: 3m（3分钟）

## ⚠️ 重要提示

1. **必须先运行Go程序获取真实数据**
   - Python脚本依赖 `test_data.json` 文件
   - 如果文件不存在，测试会失败

2. **数据获取需要网络**
   - 需要能够访问币安API
   - 如果API限制，可能需要等待

3. **数据文件可以重复使用**
   - 如果 `test_data.json` 已存在，Python脚本会使用现有数据
   - 要获取最新数据，删除文件后重新运行Go程序

## 🔧 故障排除

### 问题1: Go程序无法获取数据

**检查**:
- 网络连接
- 币安API可访问性
- 币种符号是否正确

**解决**:
```bash
# 检查网络
ping api.binance.com

# 检查币种
# 确保Symbol常量设置为正确的币种，如 "BTCUSDT"
```

### 问题2: Python脚本找不到数据文件

**解决**:
```bash
# 确保先运行Go程序
go run test_krnos_real.go

# 检查文件是否存在
ls -la test_data.json

# 然后运行Python脚本
python3 test_krnos.py
```

### 问题3: 数据不足

**可能原因**:
- 币安API返回数据少于请求数量
- 币种交易不活跃

**解决**:
- 检查币种是否正常交易
- 尝试其他币种（修改Go程序中的Symbol常量）
