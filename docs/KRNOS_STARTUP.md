# krnos预测服务启动说明

## 🚀 自动启动

krnos预测服务已集成到系统启动流程中，当运行 `./start.sh` 或直接运行 `go run main.go` 时，预测服务会自动启动。

## 📋 启动流程

### 1. 系统启动时

```
main.go 启动
  ↓
初始化配置数据库
  ↓
加载交易员配置
  ↓
启动API服务器
  ↓
启动WebSocket行情监控
  ↓
🔮 初始化krnos预测服务
  ↓
创建预测调度器
  ↓
等待5秒（让市场数据就绪）
  ↓
立即执行第一次预测
  ↓
启动定时器（每30分钟更新一次）
```

### 2. 预测执行流程

```
每30分钟执行一次：
  ↓
遍历监控的币种列表
  ↓
获取市场数据
  ↓
获取历史价格和成交量数据
  ↓
判断当前趋势
  ↓
运行krnos模型预测（10次蒙特卡罗模拟）
  ↓
保存预测结果到缓存
  ↓
等待下次更新
```

## ⚙️ 配置说明

### 监控币种

预测服务会自动监控以下币种（按优先级）：

1. **自定义币种**：如果数据库中有配置的自定义币种，使用自定义币种
2. **默认币种**：如果未配置自定义币种，使用 `config.json` 中的默认币种
3. **基础币种**：如果都没有，至少监控 BTC 和 ETH

### 预测更新策略

- **首次预测**：系统启动后5秒执行（等待市场数据就绪）
- **定时更新**：每30分钟自动更新一次
- **差异触发**：当预测结果与当前市场有较大差异时，自动重新预测

预测执行时会看到：

```
✓ BTCUSDT 预测完成
✓ ETHUSDT 预测完成
🔮 krnos预测运行完成
```

## 🔍 验证预测服务

### 1. 检查日志

运行系统后，查看日志确认预测服务已启动：

```bash
# 如果使用Docker
./start.sh logs

# 如果直接运行
# 查看终端输出
```

### 2. 检查预测缓存

预测结果保存在 `./predictor_cache/` 目录：

```bash
ls -la predictor_cache/
# 应该看到 latest_prediction.json 和按时间戳命名的预测文件
```

### 3. 查看预测内容

```bash
cat predictor_cache/latest_prediction.json
```

## ⚠️ 注意事项

1. **首次启动**：首次启动时，预测服务会在5秒后执行第一次预测
2. **市场数据依赖**：预测需要市场数据，确保WebSocket行情监控正常运行
3. **历史数据**：当前实现使用临时数据，需要从market包获取真实历史数据
4. **模型依赖**：需要先设置etg_ai项目并下载krnos模型

## 🛠️ 故障排除

### 问题1: 预测服务未启动

**检查**：
- 查看日志是否有 "🔮 初始化krnos预测服务..." 消息
- 检查是否有错误信息

**解决**：
- 确保Python环境正常
- 检查etg_ai项目是否已设置

### 问题2: 预测失败

**检查**：
- 查看日志中的错误信息
- 检查市场数据是否正常

**解决**：
- 确保市场数据服务正常运行
- 检查历史数据获取是否正常

### 问题3: 预测结果不准确

**检查**：
- 查看预测缓存文件
- 检查历史数据质量

**解决**：
- 确保历史数据充足（至少20个数据点）
- 检查模型是否正确加载
