# Trend Following 策略完整分析报告

## 一、整体流程

### 1.1 交易周期流程（每5分钟/3分钟）

```
Run() 主循环
  └─> runCycle() 每个周期执行
      ├─> 1. 检查风险控制（stopUntil）
      ├─> 2. 重置日盈亏（24小时重置）
      ├─> 3. 检测持仓变化（自动记录止损/止盈触发的平仓）
      ├─> 3.5. 自动移动止损检查（checkAndAdjustTrailingStopLoss）
      │   ├─> 动态止盈检查（基于回撤）
      │   ├─> 回撤保护检查（调整止损）
      │   └─> 常规移动止损检查（基于盈利阶段）
      ├─> 4. 构建交易上下文（buildTradingContext）
      │   ├─> 获取账户信息
      │   ├─> 获取持仓信息
      │   ├─> 获取候选币种
      │   ├─> 获取市场数据（fetchMarketDataForContext）
      │   │   ├─> 获取K线数据（3分钟、1小时、4小时）
      │   │   ├─> 计算技术指标（EMA、MACD、RSI、ATR）
      │   │   ├─> 获取OI数据、资金费率
      │   │   ├─> 获取多空持仓比
      │   │   └─> 获取krnos预测数据
      │   └─> 分析历史表现
      ├─> 5. 调用AI获取决策（GetFullDecisionWithCustomPrompt）
      │   ├─> 构建系统提示词（包含trend_following策略）
      │   ├─> 调用MCP客户端获取AI决策
      │   └─> 验证决策（validateDecisionsWithMarketPrice）
      ├─> 6. 对决策排序（先平仓后开仓）
      ├─> 7. 执行决策（executeDecisionWithRecord）
      │   ├─> open_long/open_short：开仓并设置初始止损止盈
      │   ├─> close_long/close_short：平仓
      │   └─> hold：可能调整止损止盈
      ├─> 8. 更新持仓状态
      └─> 9. 保存决策记录
```

### 1.2 关键时间点

- **周期频率**：每5分钟（300秒）执行一次 `runCycle()`
- **数据更新**：每3分钟更新一次K线数据（通过WebSocket）
- **止损检查**：每个周期自动检查并调整移动止损
- **AI决策**：每个周期调用AI进行决策

## 二、使用的指标和数据计算

### 2.1 技术指标计算（market/data.go）

#### 2.1.1 EMA（指数移动平均线）
- **函数**：`calculateEMA(klines []Kline, period int) float64`
- **计算周期**：
  - 3分钟：EMA20（当前价格趋势）
  - 1小时：EMA20、EMA50（中期趋势）
  - 4小时：EMA20、EMA50、EMA99（长期趋势）
- **计算方法**：
  ```go
  // 初始EMA = SMA（前period个收盘价的平均值）
  // 后续EMA = (当前收盘价 - 前一个EMA) × 乘数 + 前一个EMA
  // 乘数 = 2 / (period + 1)
  ```

#### 2.1.2 MACD（移动平均收敛散度）
- **函数**：`calculateMACD(klines []Kline) float64`
- **计算周期**：3分钟、1小时、4小时
- **计算方法**：
  ```go
  // EMA12 = calculateEMA(klines, 12)
  // EMA26 = calculateEMA(klines, 26)
  // MACD = EMA12 - EMA26
  ```

#### 2.1.3 RSI（相对强弱指标）
- **函数**：`calculateRSI(klines []Kline, period int) float64`
- **计算周期**：
  - 3分钟：RSI7、RSI14
  - 1小时：RSI14
  - 4小时：RSI14
- **计算方法**（Wilder平滑方法）：
  ```go
  // 初始平均涨幅和跌幅
  // 后续使用Wilder平滑：avgGain = (avgGain × (period-1) + change) / period
  // RSI = 100 - (100 / (1 + RS))
  // RS = avgGain / avgLoss
  ```

#### 2.1.4 ATR（平均真实波幅）
- **函数**：`calculateATR(klines []Kline, period int) float64`
- **计算周期**：
  - 1小时：ATR3、ATR14
  - 4小时：ATR3、ATR14
- **计算方法**：
  ```go
  // TR = max(high-low, |high-prevClose|, |low-prevClose|)
  // 初始ATR = 前period个TR的平均值
  // 后续ATR = (ATR × (period-1) + TR) / period（Wilder平滑）
  ```

### 2.2 市场数据获取

#### 2.2.1 K线数据
- **来源**：WebSocket实时数据（WSMonitorCli）
- **周期**：3分钟、1小时、4小时
- **数量**：每个周期最近10-20根K线

#### 2.2.2 持仓量（OI）数据
- **函数**：`getOpenInterestData(symbol string) (*OIData, error)`
- **数据**：最新持仓量、平均持仓量
- **用途**：判断市场情绪、流动性过滤（<15M USD跳过）

#### 2.2.3 资金费率
- **函数**：`getFundingRateAndPrices(symbol string) (float64, float64, float64)`
- **数据**：资金费率、标记价格、指数价格
- **用途**：判断市场情绪（费率过高/过低）

#### 2.2.4 多空持仓比
- **API**：`GetLongShortRatio(symbol, "1h")`
- **数据**：多空持仓比、大户多空持仓比
- **用途**：判断市场情绪（多空力量对比）

#### 2.2.5 krnos预测数据
- **函数**：`GetPredictionForSymbol(symbol, price, trend, priceHistory, volumeHistory)`
- **数据**：
  - 40个周期的预测曲线
  - 95%置信区间
  - 趋势方向（up/down/sideways）
  - 趋势强度（0-1）
- **用途**：AI决策的重要参考（权重最高）

### 2.3 综合评分系统（信心度计算）

**基础分（60分）**：
- 所有交易的基础信心度起点

**趋势方向确认（+30分）**：
- 超长线趋势确认（+8分）：日线EMA20/EMA50、MACD、周线趋势
- krnos模型预测确认（+15分，权重最高）：
  - 预测趋势与技术指标一致且趋势方向 > 0.5：+15分
  - 预测趋势与技术指标一致且趋势方向 0.2-0.5：+10分
  - 当前价格在95%置信区间内：+8分
  - 当前价格接近置信区间下限且预测趋势为'up'：+10分
- 长线趋势确认（+7分）：4小时EMA20/EMA50、MACD
- 中期趋势确认（+5分）：1小时趋势与4小时趋势一致

**转折点识别（+15分）**：
- 3分钟价格回调至支撑位/反弹至阻力位：+5分
- 3分钟RSI7超卖/超买后反弹/回调：+5分
- 3分钟MACD金叉/死叉或柱状图转正/转负：+5分

**成交量确认（+10分）**：
- 成交量放大（>1.5倍平均成交量）：+5分
- 突破确认（价格突破关键位+成交量放大）：+5分

**其他因素（+10分）**：
- 资金费率、OI变化、多空持仓比等

**扣分项**：
- 逆日线趋势：-10分
- 预测趋势与技术指标相反：-10分
- 当前价格超出预测价格范围：-8分

**最终信心度**：基础分 + 各项加分 - 扣分

## 三、关键机制

### 3.1 风险控制机制

#### 3.1.1 交易频率控制
- **硬约束**：每小时最多2次交易
- **例外条款**：S级机会可以突破限制（需满足：信心度≥90、盈亏比≥3:1、预期盈利×杠杆>30%）
- **实现位置**：`decision/engine.go` 的 `validateDecision()` 函数

#### 3.1.2 流动性过滤
- **硬约束**：持仓价值 < 15M USD 的币种不做（多空都不做）
- **实现位置**：`decision/engine.go` 的 `fetchMarketDataForContext()` 函数
- **例外**：现有持仓必须保留（需要决策是否平仓）

#### 3.1.3 最小订单金额
- **硬约束**：订单名义价值 ≥ 20 USDT
- **实现位置**：`trader/auto_trader.go` 的 `executeOpenLongWithRecord()` 和 `executeOpenShortWithRecord()`

#### 3.1.4 保证金检查
- **检查**：开仓所需保证金 + 10%缓冲 ≤ 可用余额
- **实现位置**：`trader/auto_trader.go` 的开仓函数

### 3.2 移动止损机制（checkAndAdjustTrailingStopLoss）

#### 3.2.1 动态止盈（基于价格创新高/新低）
- **触发条件**：价格创新高（做多）或创新低（做空）时，且当前盈利 > 0
- **执行动作**：立即更新止盈单到当前价格附近（不是直接平仓）
- **策略改进**：在达到最高盈利点时立即设置止盈单，而不是等回撤后再设置
- **计算公式**：
  ```go
  // 做多：止盈价 = 当前价 × 0.998（略低于当前价，确保成交）
  // 做空：止盈价 = 当前价 × 1.002（略高于当前价，确保成交）
  ```
- **更新规则**：
  - 每次价格创新高/新低时，如果新的止盈价优于现有价格（做多时更高，做空时更低），则更新
  - 这样可以在价格回撤时及时锁定利润
- **实现位置**：`trader/auto_trader.go` 第1820-1890行（已合并到价格创新高/新低检查中）

#### 3.2.2 回撤保护（调整止损）
- **触发条件**：回撤 > 止损距离 × 1.5倍 且曾经盈利过
- **执行动作**：根据当前盈利情况调整止损
  - 盈利 ≥ 止损距离×2倍：调整至盈利的60%位置
  - 盈利 ≥ 止损距离×1倍：调整至盈利的40%位置
  - 盈利 ≥ 0但 < 止损距离×1倍：调整至保本
  - 已亏损：调整至保本
- **实现位置**：`trader/auto_trader.go` 第1965-2025行

#### 3.2.3 常规移动止损（基于盈利阶段）
- **阶段1（保本）**：盈利 ≥ 止损距离×1倍 → 止损调整至入场价
- **阶段2（锁定50%利润）**：盈利 ≥ 止损距离×2倍 → 止损调整至盈利的50%位置
- **阶段3（锁定70%利润）**：盈利 ≥ 止损距离×3倍 → 止损调整至盈利的70%位置
- **阶段4（锁定75-85%利润）**：
  - 盈利 ≥ 止损距离×4倍 → 止损调整至盈利的75%位置
  - 盈利 ≥ 止损距离×5倍 → 止损调整至盈利的80%位置
  - 盈利 ≥ 止损距离×6倍 → 止损调整至盈利的85%位置
- **实现位置**：`trader/auto_trader.go` 第2035-2118行

#### 3.2.4 价格创新高/新低时的重新计算
- **触发条件**：价格创新高（做多）或创新低（做空）
- **执行动作**：重新计算并设置止损（基于当前盈利阶段）
- **实现位置**：`trader/auto_trader.go` 第1952-2034行

### 3.3 持仓变化检测机制

#### 3.3.1 自动检测止损/止盈触发
- **方法**：对比当前持仓与上一个周期的持仓
- **判断逻辑**：
  - 如果上一个周期有持仓，当前周期没有 → 已平仓
  - 根据盈亏判断是止损还是止盈触发
- **实现位置**：`trader/auto_trader.go` 的 `detectAndRecordPositionChanges()` 函数

### 3.4 决策验证机制

#### 3.4.1 盈亏比验证
- **硬约束**：盈亏比 ≥ 1.3:1
- **计算方法**（使用实际市场价格）：
  ```go
  // 风险百分比 = |入场价 - 止损价| / 入场价 × 100%
  // 收益百分比 = |止盈价 - 入场价| / 入场价 × 100%
  // 考虑交易费用（0.04%开仓 + 0.04%平仓 = 0.08%）
  // 实际风险 = 风险百分比 + 0.08%
  // 实际收益 = 收益百分比 - 0.08%
  // 盈亏比 = 实际收益 / 实际风险
  ```
- **实现位置**：`decision/engine.go` 的 `validateDecision()` 函数

#### 3.4.2 止盈距离验证
- **硬约束**：
  - 普通杠杆（<15x）：止盈价格必须距离入场价格至少 1.5%
  - 高杠杆（≥15x）：止盈价格必须距离入场价格至少 2.0%
- **实现位置**：`decision/engine.go` 的 `validateDecision()` 函数

#### 3.4.3 止损距离验证
- **建议范围**：2-3%（基于ATR计算）
- **实现位置**：`decision/engine.go` 的 `validateDecision()` 函数

## 四、止盈和止损计算

### 4.1 初始止损计算（AI决策时）

#### 4.1.1 基于ATR计算
- **公式**：
  ```
  做多止损 = 入场价 - (ATR × 倍数)
  做空止损 = 入场价 + (ATR × 倍数)
  ```
- **倍数**：通常为1.5-2.5（根据波动性调整）
- **参考周期**：4小时ATR14（权重10%）、1小时ATR3（权重5%）

#### 4.1.2 基于关键支撑/阻力位
- **做多止损**：设置在关键支撑位下方，或入场价 - (ATR × 1.5-2.5)
- **做空止损**：设置在关键阻力位上方，或入场价 + (ATR × 1.5-2.5)
- **优先级**：优先使用关键支撑/阻力位，否则使用ATR计算

#### 4.1.3 止损距离范围
- **建议范围**：2-3%
- **高波动币种**：可以使用3%止损
- **低波动币种**：可以使用1.5-2%止损

### 4.2 初始止盈计算（AI决策时）

#### 4.2.1 基于盈亏比
- **硬约束**：盈亏比 ≥ 1.3:1
- **计算公式**：
  ```
  做多止盈 = 入场价 + (止损距离 × 1.3)
  做空止盈 = 入场价 - (止损距离 × 1.3)
  ```
- **示例**：止损2%，止盈至少2.6%

#### 4.2.2 基于关键支撑/阻力位
- **做多止盈**：设置在关键阻力位
- **做空止盈**：设置在关键支撑位
- **原则**：不要过早止盈，让利润奔跑

#### 4.2.3 止盈距离要求
- **硬约束**：
  - 普通杠杆（<15x）：止盈价格必须距离入场价格至少 1.5%
  - 高杠杆（≥15x）：止盈价格必须距离入场价格至少 2.0%

### 4.3 动态止盈计算（每5分钟检查）

#### 4.3.1 触发条件
- **回撤阈值**：从最高盈利点回撤 ≥ 2%
- **当前盈利**：必须 > 0（仍盈利）

#### 4.3.2 计算公式
```go
// 做多：止盈价 = 当前价 × 0.998（略低于当前价，确保成交）
// 做空：止盈价 = 当前价 × 1.002（略高于当前价，确保成交）
```

#### 4.3.3 更新频率控制
- **避免频繁更新**：如果新价格与已设置的动态止盈价差异 < 0.5%，不更新

### 4.4 移动止损计算（每5分钟检查）

#### 4.4.1 基于盈利阶段
- **阶段1（保本）**：
  ```
  新止损价 = 入场价
  ```
- **阶段2（锁定50%利润）**：
  ```
  做多：新止损价 = 入场价 + (当前价 - 入场价) × 0.5
  做空：新止损价 = 入场价 - (入场价 - 当前价) × 0.5
  ```
- **阶段3（锁定70%利润）**：
  ```
  做多：新止损价 = 入场价 + (当前价 - 入场价) × 0.7
  做空：新止损价 = 入场价 - (入场价 - 当前价) × 0.7
  ```
- **阶段4（锁定75-85%利润）**：
  ```
  做多：新止损价 = 入场价 + (当前价 - 入场价) × 0.75/0.80/0.85
  做空：新止损价 = 入场价 - (入场价 - 当前价) × 0.75/0.80/0.85
  ```

#### 4.4.2 基于回撤保护
- **触发条件**：回撤 > 止损距离 × 1.5倍
- **计算公式**：
  ```
  如果盈利 ≥ 止损距离×2倍：
    新止损价 = 入场价 + (当前价 - 入场价) × 0.6
  如果盈利 ≥ 止损距离×1倍：
    新止损价 = 入场价 + (当前价 - 入场价) × 0.4
  如果盈利 ≥ 0但 < 止损距离×1倍：
    新止损价 = 入场价（保本）
  如果已亏损：
    新止损价 = 入场价（保本）
  ```

#### ⚠️ 需要注意的地方

1. **动态止盈和初始止盈的存储**
   - **初始止盈**：存储在 `positionInitialTakeProfit[posKey]`
   - **动态止盈**：存储在 `positionInitialTakeProfit[posKey+"_dynamic"]`
   - **问题**：使用同一个map存储，但key不同，不会冲突
   - **建议**：可以考虑使用单独的map存储动态止盈价格

2. **移动止损检查的优先级**
   - **优先级1**：动态止盈（更新止盈单）
   - **优先级2**：回撤保护（调整止损）
   - **优先级3**：常规移动止损（基于盈利阶段）
   - **优先级4**：价格创新高/新低时的重新计算
   - **当前实现**：按顺序检查，符合优先级要求

3. **止损止盈设置的冲突**
   - **问题**：交易所可能只允许一个止损单和一个止盈单
   - **当前处理**：
     - 动态止盈会替换初始止盈单（但逻辑上保留初始止盈目标）
     - 移动止损会替换旧的止损单
   - **建议**：确保交易所API支持替换订单，否则需要先取消再设置

### 5.2 数据计算的一致性

#### ✅ 一致的计算方式
- **盈利百分比计算**：
  - 移动止损使用价格百分比：`(当前价 - 入场价) / 入场价 × 100%`
  - 持仓盈亏使用保证金百分比：`未实现盈亏 / 保证金 × 100%`
  - **说明**：两种计算方式用途不同，不冲突

- **止损距离计算**：
  - 统一使用价格百分比：`|入场价 - 初始止损价| / 入场价 × 100%`

### 5.3 潜在问题

1. **动态止盈和初始止盈的冲突**
   - **问题**：动态止盈会替换初始止盈单，但逻辑上应该保留初始止盈目标
   - **当前处理**：记录初始止盈价格，但实际止盈单被替换
   - **影响**：如果价格继续上涨，可能无法达到初始止盈目标
   - **建议**：考虑实现分批止盈，保留部分仓位到初始止盈目标

2. **移动止损和动态止盈的协调**
   - **问题**：移动止损调整止损价格，动态止盈调整止盈价格，两者可能冲突
   - **当前处理**：先检查动态止盈，再检查移动止损，顺序合理
   - **建议**：确保两者不会同时触发导致订单冲突

## 六、总结

### 6.1 策略特点

1. **多周期分析**：结合3分钟、1小时、4小时多个时间周期
2. **技术指标丰富**：EMA、MACD、RSI、ATR等多种指标
3. **AI决策**：使用大模型进行综合分析和决策
4. **风险控制严格**：交易频率、流动性、保证金等多重检查
5. **动态管理**：移动止损、动态止盈、回撤保护等机制

### 6.2 关键机制

1. **移动止损**：根据盈利阶段自动调整止损，锁定利润
2. **动态止盈**：基于回撤自动更新止盈单，及时锁定利润
3. **回撤保护**：防止利润大幅回撤，主动保护已获得利润
4. **持仓变化检测**：自动检测止损/止盈触发的平仓

### 6.3 建议改进

1. **分批止盈**：实现分批止盈机制，保留部分仓位到初始止盈目标
2. **动态止盈存储**：使用单独的map存储动态止盈价格，避免混淆
3. **订单冲突处理**：确保交易所API支持替换订单，否则需要先取消再设置
4. **日志完善**：增加更详细的日志，便于追踪止损止盈的调整过程
