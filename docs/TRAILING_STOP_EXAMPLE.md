# 动态止盈策略详细示例

## 策略概述

动态止盈（移动止损）是一个**自动执行的保护机制**，系统在每个决策周期（每3分钟）自动检查所有持仓的盈利情况，并根据预设规则自动调整止损价格，实现：
- ✅ **让利润继续奔跑**：在趋势延续时，保留部分利润继续增长
- ✅ **锁定已获利润**：在盈利达到阈值时，逐步锁定利润
- ✅ **防止大幅回撤**：通过回撤保护机制，及时保护利润

## 完整交易示例：ETHUSDT 做多交易

### 初始状态（开仓时 - T0）

**交易参数**：
```
币种: ETHUSDT
方向: 做多 (LONG)
入场价格: 3,000 USDT
初始止损: 2,940 USDT
止损距离: (3,000 - 2,940) / 3,000 = 2%
持仓数量: 1 ETH
杠杆: 10x
保证金: (1 × 3,000) / 10 = 300 USDT
```

**系统记录**：
```go
positionInitialStopLoss["ETHUSDT_long"] = 2,940
positionCurrentStopLoss["ETHUSDT_long"] = 2,940
positionHighestProfit["ETHUSDT_long"] = 0.0
```

**关键点**：
- 止损距离是 **2%**（价格百分比），与杠杆无关
- 如果价格跌到 2,940，亏损是 2%（价格），但相对于保证金是 20%（2% × 10x）

## 阶段1：保本阶段（T1 - 3分钟后）

### 价格变化
```
当前价格: 3,060 USDT
价格盈利: (3,060 - 3,000) / 3,000 = 2%
```

### 系统检查逻辑

**步骤1：计算当前盈利**
```go
currentProfitPct = ((markPrice - entryPrice) / entryPrice) * 100
                 = ((3,060 - 3,000) / 3,000) * 100
                 = 2.0%
```

**步骤2：计算止损距离**
```go
stopLossDistancePct = ((entryPrice - initialStopLoss) / entryPrice) * 100
                    = ((3,000 - 2,940) / 3,000) * 100
                    = 2.0%
```

**步骤3：判断是否触发移动止损**
```
当前盈利 2% = 止损距离 2% × 1倍
→ 触发阶段1（保本）
```

**步骤4：计算新止损价**
```go
newStopLoss = entryPrice  // 保本
            = 3,000 USDT
```

**步骤5：执行调整**
```go
// 检查：新止损价 3,000 > 当前止损价 2,940（做多时只能上移）
if (side == "long" && newStopLoss > currentStopLoss) {
    trader.SetStopLoss("ETHUSDT", "LONG", 1.0, 3,000)
    positionCurrentStopLoss["ETHUSDT_long"] = 3,000
}
```

### 执行结果

**日志输出**：
```
✅ ETHUSDT long 移动止损已调整 阶段1（保本）: 当前盈利2.00%, 止损距离2.00%, 2940.0000 → 3000.0000
```

**系统状态更新**：
```go
positionCurrentStopLoss["ETHUSDT_long"] = 3,000
positionHighestProfit["ETHUSDT_long"] = 2.0
```

**盈亏分析**：
- 价格盈利：2%
- 相对于保证金盈利：2% × 10x = **20%**
- 止损已调整至保本，即使价格回撤到 3,000 也不会亏损本金


**步骤1：计算当前盈利**

**步骤2：更新最高盈利记录**
```go

    positionHighestProfit["ETHUSDT_long"] = 4.0

```

**步骤3：判断是否触发移动止损**

**步骤4：计算新止损价**
```go

            = 3,000 + (3,120 - 3,000) * 0.5
            = 3,000 + 120 * 0.5
            = 3,000 + 60
```

**步骤5：执行调整**
```go
// 检查：新止损价 3,060 > 当前止损价 3,000（做多时只能上移）

```

### 执行结果

**日志输出**：
```
✅ ETHUSDT long 移动止损已调整 阶段2（锁定50%利润）: 当前盈利4.00%, 止损距离2.00%, 3000.0000 → 3060.0000
```

**系统状态更新**：
```go

positionHighestProfit["ETHUSDT_long"] = 4.0
```

**盈亏分析**：
- 价格盈利：4%
- 相对于保证金盈利：4% × 10x = **40%**
- **已锁定 2% 价格利润**（从 3,000 到 3,060，相对于保证金是 20%）
- **剩余 2% 价格利润继续奔跑**（从 3,060 到 3,120，相对于保证金是 20%）

**关键点**：
- 这是**第一个关键保护点**，必须立即执行
- 即使价格回撤到 3,060，也能锁定 2% 利润（20% 保证金利润）

## 回撤保护机制触发（T7 - 21分钟后）

### 价格开始回撤
```
当前价格: 3,240 USDT（从最高点 3,360 回撤）
当前盈利: (3,240 - 3,000) / 3,000 = 8%
最高盈利记录: 12.0%
回撤: 12.0% - 8.0% = 4%
```

### 系统检查逻辑（回撤保护优先级最高）

**步骤1：计算回撤**
```go
highestProfit = positionHighestProfit["ETHUSDT_long"]  // 12.0%
currentProfitPct = 8.0%
drawdown = highestProfit - currentProfitPct
         = 12.0% - 8.0%
         = 4.0%
```

**步骤2：计算回撤阈值**
```go
stopLossDistancePct = 2.0%
drawdownThreshold = stopLossDistancePct * 1.5
                   = 2.0% * 1.5
                   = 3.0%
```

**步骤3：判断是否触发回撤保护**
```
回撤 4% > 回撤阈值 3%
→ 触发回撤保护机制
```

**步骤4：根据当前盈利情况决定保护动作**
```
当前盈利 8% = 止损距离 2% × 4倍
当前盈利 ≥ 止损距离的2倍（4%）
→ 调整至盈利的60%位置（更保守）
```

**步骤5：计算新止损价**
```go
// 做多：新止损价 = 入场价 + (当前价 - 入场价) × 0.6
newStopLoss = entryPrice + (markPrice - entryPrice) * 0.6
            = 3,000 + (3,240 - 3,000) * 0.6
            = 3,000 + 240 * 0.6
            = 3,000 + 144
            = 3,144 USDT
```

**步骤6：执行调整**
```go
// 检查：新止损价 3,144 > 当前止损价 3,306？
// 不！3,144 < 3,306，所以不调整（止损只能上移，不能下移）
```

### 执行结果

**日志输出**：
```
⚠️ 回撤保护触发，但新止损价3,144低于当前止损价3,306，不调整（止损只能上移）
```

**关键点**：
- 回撤保护已触发，但由于当前止损价（3,306）已经很高，系统不会下调止损
- 这是正确的行为：**止损只能向有利方向移动**（做多时只能上移）

## 策略总结

### 关键特点

1. **自动化执行**：系统每3分钟自动检查并调整，无需AI决策
2. **基于价格百分比**：所有计算基于价格变化，与杠杆无关
3. **渐进式保护**：随着盈利增加，逐步提高止损位
4. **回撤保护优先**：回撤保护机制优先级最高，及时保护利润
5. **只能上移**：止损只能向有利方向移动，不能下调

### 各阶段总结

| 阶段 | 触发条件 | 锁定比例 | 示例（止损2%） |
|------|---------|---------|---------------|
| 阶段1 | 盈利 ≥ 止损距离 × 1倍 | 保本（0%） | 盈利2% → 止损至入场价 |
| 阶段2 | 盈利 ≥ 止损距离 × 2倍 | 锁定50% | 盈利4% → 锁定2% |
| 阶段3 | 盈利 ≥ 止损距离 × 3倍 | 锁定70% | 盈利6% → 锁定4.2% |
| 阶段4 | 盈利 ≥ 止损距离 × 4倍 | 锁定75% | 盈利8% → 锁定6% |
| 阶段4 | 盈利 ≥ 止损距离 × 5倍 | 锁定80% | 盈利10% → 锁定8% |
| 阶段4 | 盈利 ≥ 止损距离 × 6倍 | 锁定85% | 盈利12% → 锁定10.2% |

### 回撤保护总结

| 当前盈利 | 保护动作 | 锁定比例 |
|---------|---------|---------|
| ≥ 止损距离 × 2倍 | 调整至盈利60% | 更保守 |
| ≥ 止损距离 × 1倍 | 调整至盈利40% | 锁定更多 |
| < 止损距离 × 1倍 | 调整至保本 | 入场价 |

### 优势

1. ✅ **让利润奔跑**：在趋势延续时，保留部分利润继续增长
2. ✅ **锁定利润**：在盈利达到阈值时，逐步锁定利润
3. ✅ **防止回撤**：通过回撤保护，及时保护已获利润
4. ✅ **自动化**：无需人工干预，系统自动执行
5. ✅ **适应杠杆**：基于价格百分比，适用于任何杠杆倍数
