# 币安历史数据获取说明

## 📊 数据获取实现

### 1. 历史数据客户端 (`market/history_client.go`)

创建了专门的历史数据客户端，处理币安API的限制：

- **币安API限制**：单次请求最多1000条K线数据
- **自动分批获取**：如果需要的数据超过1000条，自动分批获取并合并
- **数据验证**：确保数据是最新的，验证数据时间戳
- **错误处理**：完善的错误处理和日志记录

### 2. 关键功能

#### GetHistoricalKlines
获取历史K线数据，自动处理API限制：

```go
klines, err := historyClient.GetHistoricalKlines(symbol, "3m", 200, nil)
```

- 如果请求数量 ≤ 1000：直接获取
- 如果请求数量 > 1000：自动分批获取并合并

#### GetLatestKlinesForPrediction
专门用于预测的数据获取方法：

```go
klines, err := historyClient.GetLatestKlinesForPrediction(symbol, "3m", 200)
```

特性：
- 自动获取最新数据（不指定endTime）
- 验证数据是否足够（至少100条）
- 验证数据是否是最新的（检查最后一条K线的时间）
- 返回按时间从旧到新排序的数据

### 3. 数据验证

#### 数据量验证
- **最小数据点**：至少100条（`MinDataPointsForPrediction`）
- **最佳数据点**：200条（`OptimalDataPointsForPrediction`）
- 如果数据不足，会记录警告并跳过预测

#### 数据新鲜度验证
- 检查最后一条K线的时间戳
- 根据K线间隔判断数据是否过时
- 允许最多2个间隔的延迟（例如3分钟K线，允许6分钟延迟）

#### 数据一致性验证
- 验证价格和成交量数据长度是否一致
- 如果不一致，使用较短的长度并记录警告

## 🔍 数据获取流程

```
1. 调用 GetLatestKlinesForPrediction
   ↓
2. 检查请求数量是否超过1000
   ↓
3. 如果超过，分批获取：
   - 第一次：获取最新1000条
   - 第二次：获取更早的1000条（使用endTime）
   - 合并数据
   ↓
4. 验证数据量是否足够
   ↓
5. 验证数据是否是最新的
   ↓
6. 返回K线数据
```

## ⚠️ 注意事项

### 1. API限制
- 币安API单次请求最多1000条
- 如果超过，会自动分批获取
- 每次请求之间有100ms延迟，避免请求过快

### 2. 数据准确性
- 使用3分钟K线数据（与交易系统保持一致）
- 确保数据是最新的（验证时间戳）
- 验证数据完整性（价格和成交量长度一致）

### 3. 错误处理
- 如果获取失败，记录错误并跳过该币种
- 如果数据不足，记录警告并跳过预测
- 如果数据过时，记录警告但继续使用

## 📝 使用示例

### 在预测调度器中使用

```go
// 获取历史价格数据
priceHistory, err := ps.getPriceHistory(symbol, 200)
if err != nil {
    log.Printf("获取历史价格数据失败: %v", err)
    continue
}

// 获取历史成交量数据
volumeHistory, err := ps.getVolumeHistory(symbol, 200)
if err != nil {
    log.Printf("获取历史成交量数据失败: %v", err)
    continue
}

// 验证数据
if len(priceHistory) < 100 || len(volumeHistory) < 100 {
    log.Printf("数据不足，跳过预测")
    continue
}
```

## 🎯 数据质量保证

1. **真实性**：直接从币安API获取，不使用临时数据
2. **即时性**：获取最新数据，验证时间戳
3. **完整性**：处理API限制，确保获取足够的数据
4. **一致性**：验证价格和成交量数据长度一致

## 🔧 故障排除

### 问题1: 数据获取失败

**可能原因**：
- 网络问题
- 币安API限制
- 币种不存在

**解决**：
- 检查网络连接
- 查看日志中的错误信息
- 验证币种符号是否正确

### 问题2: 数据不足

**可能原因**：
- 币种交易不活跃
- API返回数据少于请求数量

**解决**：
- 检查币种是否正常交易
- 查看日志中的警告信息
- 考虑使用更少的数据点

### 问题3: 数据过时

**可能原因**：
- 币安API延迟
- 网络延迟

**解决**：
- 检查网络连接
- 查看日志中的警告信息
- 如果延迟过大，考虑重新获取
